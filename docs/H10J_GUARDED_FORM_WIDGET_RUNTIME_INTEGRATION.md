# H10J Guarded Form-Widget Runtime Integration

## Scope

H10J integrates the guarded `PDF/UA-1/7.18.4` form-widget repair into the orchestrator behind an explicit opt-in flag.

```text
target rule: PDF/UA-1/7.18.4
strategy id: form_widget_structure_construction_v1
repair script: tools/repair/repair_form_widget_structure.py
repair version: 1.4.0
orchestrator flag: --enable-guarded-form-widget-repair
default runtime behavior changed: false
```

H10J does not add the guarded repair to active rule-map strategies and does not make the repair production-default.

## Implemented behavior

```text
guarded lookup precondition generated by orchestrator: true
lookup --enable-guarded-candidates passed only under explicit orchestrator flag: true
lookup --precondition-report passed only under explicit orchestrator flag: true
guarded repair step removed from normal deterministic repair loop: true
guarded repair runtime executed only as dedicated guarded runtime: true
candidate output written outside workspace/jobs: true
candidate promotion requires guarded acceptance promote_candidate_to_final: true
```

Safe intermediate candidate path:

```text
/app/workspace/guarded_candidates/<job>/form_widget_structure/output.pdf
```

## Interpretation

The smoke is successful because it proves the guarded path executes and fails closed. It does not need to produce PASS.

The guarded candidate cleared the target rule in validation evidence, but guarded acceptance rejected the candidate because the post form-widget structure diagnostic did not produce a PASS-equivalent result. The package remained report-only and the final authoritative status preserved ESCALATION.

## Boundary

This H10J smoke is Docker CLI runtime evidence only. It is not Open WebUI E2E production-path evidence.

Production readiness is not claimed until a user-facing Open WebUI prompt beginning with `PDF:` routes through Hermes, invokes `/app/tools/orchestrate/remediate.py`, and truthfully reports the resulting `STATUS.json`, `orchestrator_outcome.json`, and deliverables package.
